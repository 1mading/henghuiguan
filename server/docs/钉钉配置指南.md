# 恒慧管 · 钉钉配置指南

参考 FactoryCheckList 的配置方式，**不改动** `恒慧管.html` 业务内容；通过小程序壳 + 后端 `.env` 对接钉钉。

---

## 一、目录说明

| 路径 | 作用 |
|------|------|
| `server/.env` | 后端钉钉凭证（**勿提交 Git**） |
| `server/miniapp/config.js` | 小程序 API 地址、CorpId（**勿提交 Git**） |
| `server/miniapp/config.js.example` | 配置模板 |
| `server/miniapp/` | 钉钉小程序工程（开发者工具打开此目录） |
| `server/恒慧管/` | 钉钉官方 Demo，**可忽略** |

---

## 二、你需要填写的凭证（3 处保持一致）

在 [钉钉开放平台](https://open.dingtalk.com/) 创建**企业内部小程序**后，记录：

| 凭证 | 填到哪里 |
|------|----------|
| CorpId | `server/.env` → `DINGTALK_CORP_ID`<br>`server/miniapp/config.js` → `DINGTALK_CORP_ID` |
| AppKey | `server/.env` → `DINGTALK_APP_KEY` |
| AppSecret | `server/.env` → `DINGTALK_APP_SECRET`（**仅后端**） |
| AgentId | `server/.env` → `DINGTALK_AGENT_ID` |
| miniAppId | `server/.env` → `DINGTALK_MINI_APP_ID` |
| HTTPS 域名 | 开放平台 **HTTP 安全域名**<br>`server/.env` → `PUBLIC_BASE_URL`<br>`server/miniapp/config.js` → `API_BASE_URL` |
| API Key | `server/.env` → `API_KEY`<br>`server/miniapp/config.js` → `API_KEY`（与 FactoryCheckList 一致） |

## 三、本地调试（5 步）

### 1. 启动后端

```bash
cd server
npm install
npm start
```

浏览器可访问：http://localhost:3000/恒慧管.html

### 2. 配置小程序（已生成默认本地配置）

`server/miniapp/config.js` 默认：

- `SKIP_LOGIN_CHECK: true`（跳过钉钉 getAuthCode，用 `DEV_MOCK_DINGTALK_USER_ID` 调 profile）
- `API_BASE_URL: http://192.168.x.x:3000/api`（真机须与电脑同网）
- `API_KEY: henghuiguan-dev-key`（与 `.env` 一致）

### 3. 打开钉钉开发者工具

- **导入项目** → 目录选：`server/miniapp`
- 已启用 `.mini-ide/project-ide.json` 中 `ignoreHttpDomainCheck`（仅本地）

### 4. 编译运行

小程序启动流程（与 FactoryCheckList 一致）：

1. `getAuthCode` → `POST /api/auth/profile`（本地 SKIP 模式则仅带 `X-User-Id`）
2. 请求头携带 `X-Api-Key` + `X-User-Id`
3. 原生首页展示任务概览，可进入「我的任务」「项目列表」

PC 浏览器仍可直接打开 `恒慧管.html`（H5 完整功能）。

### 5. 上线前切换

`server/miniapp/config.js`：

```javascript
SKIP_LOGIN_CHECK: false,
DINGTALK_CORP_ID: '你的CorpId',
API_BASE_URL: 'https://你的域名/api',
API_KEY: '你的生产密钥',
```

`server/.env`：

```env
ALLOW_DEMO_LOGIN=false
PUBLIC_BASE_URL=https://你的域名
API_KEY=你的生产密钥
# 填齐 DINGTALK_* 
```

## 七、常见问题

| 现象 | 处理 |
|------|------|
| 小程序报「无法连接」 | 确认 `npm start` 已运行；真机需 HTTPS 域名，不能用 localhost |
| error 4/11 安全域名 | 开放平台添加域名并重新上传小程序 |
| 未授权页 | 在 users 表补充该员工 dingTalkUserId |
| web-view 空白 | 小程序已改为原生页；H5 请浏览器直接打开 |
| 知识库选择器报 `Wiki.Node.Read` | 见 **第八节**，开通权限并重新发布应用 |

## 八、钉钉文档附件（知识库选择器）

项目/任务附件支持从钉钉知识库选择在线文档，后端会调用 Wiki API，**必须**开通以下权限：

| 权限名称 | Scope | 用途 |
|----------|-------|------|
| 知识库节点读 | `Wiki.Node.Read` | 列出知识库、浏览目录、解析文档链接 |

### 开通步骤

1. 打开 [钉钉开放平台](https://open.dingtalk.com/) → **应用开发** → 进入恒慧管对应应用  
2. **权限管理** → 搜索「知识库」或 `Wiki.Node.Read`  
3. 勾选 **知识库节点读**，提交申请（企业管理员审批）  
4. 也可直接打开接口返回的一键申请链接（形如 `https://open-dev.dingtalk.com/appscope/apply?content=...Wiki.Node.Read`）  
5. 审批通过后 **重新发布** 应用，等待 **1～5 分钟** 生效  
6. 重启恒慧管后端（刷新 access_token），刷新 H5 页面再试  

### 说明

- 左侧能列出知识库、右侧目录为空，通常是 **节点读权限** 未生效，完成上述步骤即可。  
- 只能添加 **人员档案中至少一人有查看权限** 的知识库文档（系统会汇总档案内全部成员的可见知识库）。  
- 浏览某知识库目录时，会自动选用档案中对该库有权限的成员身份调用钉钉 API。  
- 可选：在 `server/.env` 设置 `DINGTALK_WIKI_WORKSPACE_KEYWORDS=关键词` 对汇总结果再做名称过滤。  
- 同步通讯录时会写入 `dingTalkUnionId`，若列表为空请先同步通讯录。
