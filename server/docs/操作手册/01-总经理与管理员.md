# 总经理 / 管理员 操作手册

> 适用角色：`gm`（总经理）、`admin`（管理员）  
> 权限范围：**全公司** 所有部门的项目、任务、人员

---

## 1. 您能看到什么

- 全部部门的项目与任务（含临时任务）
- 全员操作日志、推送记录（人员档案页底部）
- 全公司团队饱和度与成员看板

---

## 2. 菜单与职责

| 菜单 | 主要用途 |
|------|----------|
| **我的待办** | **本人** 负责/协办末级任务；**部门** 页签按部门监督 |
| **项目** | 新建/编辑/归档全公司项目，管理项目文档 |
| **新建事项** | 创建临时事项（可选挂项目） |
| **团队管理**（更多） | 各部门成员饱和度、成员任务看板 |
| **人员档案**（更多） | 维护人员、**同步钉钉通讯录** |
| **操作日志**（更多） | 全量变更溯源 |
| **归档管理**（更多） | 查询已归档项目 |
| **数据导入**（更多） | 批量导入项目/任务数据 |
| **更新记录**（更多） | 查看系统版本更新说明 |

## 3. 典型操作流程

### 3.1 首次上线：同步钉钉通讯录

1. 从钉钉工作台登录恒慧管
2. 进入 **人员档案** → **同步钉钉**
3. 选择 **导入/替换**，勾选要纳入系统的部门
4. 先点 **预览**，核对「新增 / 更新 / 停用」名单
5. 确认无误后点 **确认替换**

> **导入/替换**：以钉钉名册为准写入档案；本地姓名与钉钉姓名**模糊匹配**时，改为钉钉最新显示名，并同步更新任务负责人、项目负责人、协办人等引用。勾选部门内本地多出的人会**软停用**（不删除）。总经理/管理员/部门经理角色不会被改成执行人员。  
> **勾选人员 / 按部门绑定**：绑定 `userid`；若姓名模糊匹配也会同步改为钉钉最新名并更新引用。

人员档案列表也可 **人为停用 / 恢复**（仅总经理/管理员；不可停用自己）。停用后不可登录、不出现在分配人选中；勾选「显示已停用」可找回。

若任务负责人在档案中**不存在或已停用**但仍有未完成任务，系统会在人员档案/我的待办提示，并钉钉通知其上级（档案中的上级、相关项目负责人；若无则通知总经理），请及时恢复人员或转办任务。

### 3.2 新建项目

1. **项目管理** → 右上角 **新建项目**
2. 填写：项目名称、描述、所属部门、负责人、计划起止日期
3. 可选指定 **团队成员**（协助推进项目内任务，可编辑/留言/上传附件，不可拆分任务或归档）
4. 保存 → 进入项目详情

### 3.3 项目状态流转

在项目详情页「项目状态」区域操作：

```
规划中 → 进行中 → 已完成 → 归档
```

- **规划中**：立项准备
- **进行中**：正式推进
- **已完成**：交付结束
- **归档**：从活跃列表移入归档管理，只读

**删除项目**（仅总经理/管理员）：项目详情或归档管理页点击 **删除**，将永久移除项目、全部关联任务及文档/附件，不可恢复。部门经理和执行人员无此权限。

### 3.4 分解任务

1. 进入 **项目详情** → **添加任务**（主任务）
2. 打开任务详情 → **添加子任务**（支持无限层级）
3. 填写负责人、计划开始、预计工时；可选填 **每日投入工时**（带教场景）；截止日期自动计算
4. 可选指定 **告知性协办人**（仅通知跟进）或 **辅助性协办人**（须填协助时段并审批，计入饱和度）
5. 保存后系统自动 **推送新任务** 给负责人；添加辅助性协办人时推送 **协办邀请**

**子任务工时规则**：同一父任务下，所有子任务预计工时之和不得超过父任务工时；保存时系统会提示调整。

### 3.5 配置跨项目任务依赖

1. 打开 **后继任务**（被卡住、须等待的那条）→ 任务详情
2. 在描述下方找到 **「依赖关系（跨项目）」** → **添加前置依赖**
3. 选择 **前置任务**（可跨项目）、阻塞强度（默认强阻塞）、说明
4. 前置未完成时：后继显示 **⏸ 阻塞中**，不占当周饱和度，不能开始/完成
5. 前置 **标记完成** 后：后继变为 **▶ 可继续**，恢复计饱和度，并推送通知

**项目级汇总**：项目详情 → **「跨项目 / 任务依赖」** 面板，查看本项目全部依赖关系。

### 3.6 我的待办 · 部门监督

1. **我的待办** → 顶部切换到 **「部门」** 页签
2. 选择 **全部部门** 或指定部门，查看该范围内成员负责的 **末级任务**
3. 点击任务进入详情，可进行驳回、转办、编辑等（须具备相应权限）
4. **「我的」** 页签仍只展示您本人负责/协办的任务

### 3.7 监督与干预任务

在任务详情页可操作：

| 按钮 | 作用 |
|------|------|
| 驳回 | 退回给负责人整改，需填原因，推送通知 |
| 转办 | 更换负责人，需填原因，推送通知 |
| 编辑 | 修改任务信息，**必填修改原因** |
| 作废 | 标记任务不再执行（需确认） |
| 标记完成 | 推进为已完成，通知项目负责人与执行人上级 |

### 3.8 上传项目文档与任务附件

1. 项目详情 → **项目文档** 区域 → 点击上传，或 **添加钉钉文档** 从知识库关联
2. 任务详情 → **任务附件** 区域 → 点击上传、**Ctrl+V 粘贴截图**，或添加钉钉文档链接
3. 支持在线预览/下载/删除（需权限）；图片类附件可点击放大查看

### 3.9 任务留言与 @ 提醒

1. 打开任务详情，滚动至 **留言讨论**
2. 输入文字；输入 **@** 从列表选择负责人、协办人或项目负责人
3. 需要附图时在留言框 **Ctrl+V 粘贴图片**（可多张），点击缩略图可放大预览
4. 点击 **发送留言**；被 @ 且已绑定钉钉 userid 的人员会收到 **【恒慧管·留言提及】** 工作通知

> 留言用于沟通过程中的说明与截图；正式交付材料请上传到 **任务附件**。

### 3.10 查看团队负荷

1. **团队管理** → 按部门筛选成员
2. 查看每人：进行中任务数、预计工时、**当周**饱和度（主责任务 + 已批准辅助性协办）；若有 **因阻塞挂起** 工时会单独显示
3. 支持按 **工作饱和度** 排序
4. 点击成员姓名 → 进入该成员任务看板
5. 顶部 **导出** 可下载团队 Excel

### 3.11 滚动大屏（会议室展示）

用于全公司项目与 **末级任务** 的滚动展示，适合会议室电视/大屏。

1. 确保服务端已启动
2. 浏览器打开（勿直接双击本地 HTML 文件）：
   - **推荐（大屏专用页）**：`http://<服务器地址>:3000/wall?key=<API_KEY>`
   - **或**：已用总经理/管理员账号登录恒慧管后，也可不带 `key` 访问（服务端校验登录态）
   - `API_KEY` 见服务端 `server/.env` 配置
3. 页面自动每 60 秒刷新；鼠标悬停可暂停滚动
4. 展示内容：全部未归档项目 + 无子任务的末级任务 + 阻塞/延期统计

> 大屏 URL 密钥勿对外泄露；会议室场景建议使用 `?key=` 方式，无需人工登录。

### 3.12 数据导出

| 页面 | 导出内容 |
|------|----------|
| 项目管理 | 项目列表 Excel |
| 任务中心 | 任务列表 Excel |
| 团队管理 | 成员饱和度 Excel |

### 3.13 数据导入

1. **数据导入** → 按模板格式准备 Excel/JSON
2. 上传并确认映射
3. 导入后检查项目与任务是否正确

## 4. 权限边界

| 您可以做 | 说明 |
|----------|------|
| 操作任意部门的任务 | 编辑、完成、驳回、转办、作废 |
| 新建任意部门项目 | 部门字段可选全部部门 |
| 编辑所有人员档案 | 含角色、部门、上级关系 |
| 同步钉钉通讯录 | **唯一** 有此菜单入口的角色 |
| 配置跨项目依赖、滚动大屏 | 全公司任务均可操作 |

| 您不需要做 | 说明 |
|------------|------|
| 修改工作日历 | 由服务端配置文件维护，见技术文档 |
| 手动触发部门日报推送 | 接口已预留，待定时任务调度 |

---

## 5. 推送您会收到什么

作为项目负责人或他人上级时，可能收到：

- **任务完成**：您负责的项目下有任务完成时
- **直属下级动态**：若您是一线人员的 `leaderId` 对应上级

分配/转办/驳回类通知主要发给任务 **执行负责人**，不一定发给总经理，除非您被指定为负责人。

## 6. 常见问题

**Q：执行人完成了任务，为什么总经理没收到推送？**  
A：完成通知发给 **项目负责人** 和 **执行人直属上级**。若您两者都不是，则不会收到。可在项目详情确认项目负责人字段。

**Q：推送显示 skipped？**  
A：接收人未绑定钉钉 userid，请在人员档案重新同步。

**Q：留言 @ 了同事但没收到钉钉？**  
A：确认对方在人员档案中已 **同步钉钉并绑定 userid**；且被 @ 人员须为当前任务的负责人、协办人或项目负责人。

**Q：如何给某人开管理员权限？**  
A：人员档案 → 编辑该用户 → 角色改为「管理员」。
