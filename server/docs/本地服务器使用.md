# 恒慧管 · 本地电脑作服务器

当前模式：**本机运行 Node，同事通过局域网 IP 访问**。

---

## 一、启动服务

双击项目根目录：

```
启动恒慧管服务器.bat
```

或命令行：

```powershell
cd server
npm start
```

启动后会显示：

- 本机：`http://localhost:3000/app`
- 局域网：`http://192.168.222.26:3000/app`（以你机器实际 IP 为准）

---

## 二、同事如何访问

1. 确保与你在**同一局域网**（同一 WiFi / 有线网段）
2. 浏览器或钉钉打开：`http://<你的IP>:3000/app`
3. 推荐从**钉钉工作台**打开（需先在钉钉后台改应用首页地址，见下文）

---

## 三、钉钉应用配置

在 [钉钉开放平台](https://open.dingtalk.com/) → 恒慧管应用：

| 配置项 | 填写 |
|--------|------|
| 应用首页地址 | `http://192.168.222.26:3000/app` |
| 服务器出口 IP | 填你电脑对外的公网 IP（若仅内网使用，按公司 IT 要求配置） |
| PC 端首页地址 | 同上 |

> IP 变化时（换网络、DHCP 重新分配），需更新 `.env` 中 `PUBLIC_BASE_URL`、`CORS_ORIGINS` 和钉钉应用首页。

---

## 四、Windows 防火墙

若同事无法访问，在本机 PowerShell（管理员）执行：

```powershell
New-NetFirewallRule -DisplayName "恒慧管 3000" -Direction Inbound -Protocol TCP -LocalPort 3000 -Action Allow
```

---

## 五、数据在哪里

| 路径 | 说明 |
|------|------|
| `server/data/henghuiguan.json` | 全部业务数据 |
| `server/data/backups/` | 自动备份（每次保存前） |

**请定期复制 `server/data/` 文件夹做备份。**

---

## 六、以后迁移到正式服务器

只需 3 步：

1. 复制整个项目（或至少 `恒慧管.html` + `server/`）到新服务器
2. 复制 `server/data/`（含备份）
3. 修改 `server/.env`：
   - `LOCAL_AS_SERVER=false`（或删除）
   - `PUBLIC_BASE_URL=https://新域名`
   - `CORS_ORIGINS=https://新域名`
4. 新服务器 `npm install` → `npm start`，钉钉应用首页改为新域名

代码和接口**无需改动**。

---

## 七、常见问题

| 现象 | 处理 |
|------|------|
| 同事打不开页面 | 检查防火墙、IP 是否正确、服务是否在跑 |
| 钉钉里无法免登 | 应用首页必须是 `http://IP:3000/app`，且手机与电脑同网段 |
| 关机后不可用 | 本地作服务器时，电脑需保持开机且服务运行 |
| IP 变了 | 改 `.env` 和钉钉首页，重启服务 |
