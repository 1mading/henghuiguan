# 通用说明（全部角色）

## 1. 登录与访问

### 正式使用（推荐）

1. 打开钉钉 → 工作台 → 找到 **恒慧管** 应用
2. 系统自动完成钉钉免登，无需输入账号密码
3. 首次使用需管理员在「人员档案」中同步钉钉通讯录，绑定 `userid`

### 演示模式（开发/内测）

- 侧边栏底部可切换不同人员账号
- 推送不会真实发送到钉钉（或走演示队列）

### 登录失败常见原因

| 现象 | 处理方式 |
|------|----------|
| 提示未找到钉钉用户 | 联系管理员同步通讯录并绑定 userid |
| 数据空白 | 检查网络，刷新页面；确认服务端已启动 |
| 登录过期 | 重新从钉钉工作台打开应用 |

## 2. 我的待办（原工作台 + 任务中心）

打开应用默认进入 **我的待办**，展示本人相关末级任务（负责人或协办人）：

- 逾期 / 临期顶部告警
- 待办 / 进行中 / 已完成列（有暂停、驳回等时才出现「其他」列）
- 部门经理 / 总经理另有 **部门** 页签，用于监督

侧栏与移动端主导航为：**待办 | 项目 | 新建 | 更多**。团队饱和度、人员档案、导入、归档、操作日志在 **更多**。

任务详情主操作：**开始 / 完成 / 留言**；转办、驳回、作废、编辑、拆子任务在 **更多**。  
编辑表单默认只展示日常字段；协办、工时、依赖在 **高级选项**（规则未改，含强制修改原因）。

> **延期与临期**：截止日期 **当天** 仍算 **临期**，不算已延期；只有截止日期 **已过**（早于今天）才显示为已延期。

---

## 3. 我的待办 vs 项目

| 页面 | 展示范围 | 典型用途 |
|------|----------|----------|
| **我的待办** | 本人负责或协办的 **末级任务**；经理+/总经理另有 **部门** 页签 | 每日推进、部门监督、导出 |
| **项目** | **全公司** 未归档项目及项目下 **全部** 任务树 | 看他人进度、项目背景、拆分/分配 |

执行人员与部门经理在项目中可 **只读** 浏览全公司项目；编辑、拆分任务、归档等须为项目负责人/创建人或总经理/管理员。

**项目团队成员**：新建/编辑项目时可指定 **团队成员**（除项目负责人外）。被指定成员对该项目下任务拥有与负责人/协办人类似的 **编辑、上传附件、留言** 权限，但 **不能** 拆分主/子任务、驳回、转办或归档项目。项目详情页会展示团队成员名单。

---

## 4. 任务类型

| 类型 | 说明 |
|------|------|
| **常规任务** | 挂在项目下，可无限层级拆分子任务 |
| **临时任务** | 不关联项目，任何角色均可快速创建，用于零星事项 |

### 任务优先级

创建或编辑任务时可设置优先级，**我的待办** 列表 **紧急** 任务会自动置顶：

| 优先级 | 说明 |
|--------|------|
| **紧急** | 需优先处理，列表置顶显示 |
| **重要** | 较高优先级 |
| **普通** | 默认 |

---

## 5. 任务状态

| 状态 | 含义 | 常见操作 |
|------|------|----------|
| 待开始 | 已分配，尚未执行 | 「开始」 |
| 进行中 | 正在推进 | 更新进度、「完成」 |
| 已完成 | 任务结束 | 可归档（经理/总经理） |
| 已暂停 | 临时搁置 | 编辑恢复 |
| 已驳回 | 被上级退回整改 | 修改后重新推进 |
| 阻塞中 | 有未完成的前置依赖，暂不能推进 | 等待前置完成；**不计入**当周饱和度 |
| 已作废 | 不再执行 | 不参与进度统计 |
| 已归档 | 项目级归档后只读 | 仅查看 |

**注意**：有 **未完成的子任务** 时，上级任务不能直接「完成」。

## 6. 计划时间与工时

| 字段 | 说明 |
|------|------|
| 计划开始日期 | 手动填写 |
| 预计工时 | 手动填写（小时） |
| **每日投入工时（选填）** | 留空按标准 **8.5 小时/天** 推算截止日；填写后按每日投入推算（如带教每天 2h、3h） |
| 计划截止日期 | **自动计算**（按服务端工作日历，跳过节假日/非上班日） |
| 实际工时 | **自动计算**（根据实际开始/结束日期与工作日历；有每日投入工时时按该值计算） |

**带教示例**：计划开始 6/18、预计 30 小时、每日投入 2 小时 → 截止日按 15 个工作日推算（而非按满日 8.5h 压缩）。

工作日历由服务端统一配置，前端不可修改。详见 [工作日历配置](../工作日历配置.md)。

## 7. 任务协办人

创建或编辑任务时可指定两类协办人（可选人员档案中的任意人员）：

| 类型 | 说明 | 工时 |
|------|------|------|
| **告知性协办** | 接收任务/项目进展通知，在我的待办中跟进 | **不计入** 饱和度 |
| **辅助性协办** | 须协办人填写可协助的开始/截止时段，经任务负责人或项目负责人 **审批** 后生效 | **计入** 当周饱和度（按协助时段分摊） |

**辅助性协办流程**：

1. 负责人/经理在任务编辑中添加辅助性协办人 → 协办人收到钉钉 **协办邀请**
2. 协办人在任务详情填写协助时段与说明 → 提交审批
3. 任务负责人或项目负责人 **同意/拒绝** → 同意后计入团队饱和度统计

## 8. 跨项目任务依赖（全部角色可见）

用于表达「须等其他任务（可跨项目）完成后，本任务才可继续」的执行顺序，与任务树 **父子拆分（parentId）** 无关，二者可同时存在。

| 概念 | 说明 |
|------|------|
| **前置依赖** | 本任务需等待指定任务 **完成后** 才可开始/完成 |
| **阻塞中** | 存在未完成前置时，任务显示 **阻塞中**；**不计入**当周工时饱和度，不算逾期/临期 |
| **标识** | 🔗 前置 · ⏸ 阻塞中 · ▶ 可继续 · 🚧 阻塞后续 · 🌐 跨项目 |

**查看入口**：

- **任务详情** →「依赖关系（跨项目）」：查看前置/后续、方案说明
- **项目详情** →「跨项目 / 任务依赖」：汇总本项目相关依赖

**配置权限**：总经理/管理员、任务负责人/可编辑人、项目负责人可 **添加/解除** 前置依赖；执行人员一般 **仅查看**。

## 9. 修改留痕

编辑已有任务时 **必须填写「修改原因」**，否则无法保存。所有变更写入操作日志，可追溯。

## 10. 钉钉工作通知与应用内消息

绑定 `userid` 后，以下事件会自动推送 **钉钉工作通知**；同时，顶部 **小喇叭** 会收到 **应用内消息通知**（未绑定 userid 也可在应用内收到）：

| 事件 | 接收人 |
|------|--------|
| 新任务分配 | 任务负责人 |
| 任务转办 | 新负责人 |
| 任务驳回 | 原负责人 |
| 即将到期 | 任务负责人（系统自动扫描） |
| 任务逾期 | 任务负责人（系统自动扫描） |
| 任务完成 | **项目负责人** + **执行人直属上级** |
| 新增任务依赖 | 后继任务负责人、相关项目负责人 |
| 前置完成解除阻塞 | 后继任务负责人 |
| 辅助性协办邀请 | 被邀请的协办人 |
| 协办时段待审批 | 任务负责人、项目负责人 |
| 协办时段已批准/已拒绝 | 协办人 |
| 留言 @ 提及 | 被 @ 的任务负责人、协办人、项目负责人 |
| 人员缺失（档案无此人/已停用但仍有未完成任务） | 上级、相关项目负责人（兜底通知总经理）；在顶栏小喇叭中展示汇总，点击可进入人员档案处理 |

推送内容分块展示：**所属项目 → 任务详情 → 操作信息**。

点击顶栏小喇叭可查看未读消息；点击某条可跳转到相关任务/项目，并可「全部已读」。约每 30 秒自动刷新未读角标。

未收到钉钉推送请检查：① 人员档案是否已绑定 userid；② 您是否为该事件的接收人（如完成通知发给项目负责人，不一定发给执行人本人）。应用内消息不依赖钉钉绑定。

## 11. 文档、附件与留言

### 11.1 项目文档与任务附件

| 位置 | 谁能上传 | 说明 |
|------|----------|------|
| 项目详情 · 项目文档 | 总经理/管理员、项目负责人 | 需求书、方案等；支持上传文件或 **添加钉钉文档链接** |
| 任务详情 · 任务附件 | 负责人、协办人、项目负责人、管理员 | 截图、交付材料等；支持上传、Ctrl+V 粘贴、钉钉文档链接 |

**粘贴图片**：在任务详情 **附件区域** 点击后，按 **Ctrl+V** 可直接粘贴剪贴板中的图片（需已登录并连接服务端）。

**钉钉文档**：点击「添加钉钉文档」可从团队知识库/我的文档中选择并关联，无需重复上传文件。

需连接服务端；单文件大小限制见服务端配置（默认 20MB）。

### 11.2 任务留言讨论

任务详情底部 **留言讨论** 用于项目相关人员在任务上下文中沟通进度与问题。

| 项目 | 说明 |
|------|------|
| **谁可以留言** | 任务负责人、协办人、项目负责人；总经理/管理员 |
| **@ 提及** | 输入 `@` 弹出可选人员（负责人、协办人、项目负责人），选中后发送；被 @ 且已绑定 userid 的人员会收到钉钉 **留言提及** 通知 |
| **粘贴图片** | 在留言输入框内 **Ctrl+V** 粘贴图片，发送前可预览；**点击缩略图可放大查看** |
| **纯图片留言** | 可不填文字，仅发送图片 |

> 留言图片保存在留言记录中，不会混入任务附件列表。任务附件用于正式交付材料，留言用于过程沟通。

## 12. 系统更新记录

侧边栏 **「更新记录」** 全员可见，用于查看各版本功能变更说明。

| 项目 | 说明 |
|------|------|
| **登录提醒** | 有新版本且您尚未阅读时，登录后会弹出更新说明；点击「知道了」标记为已读 |
| **历史版本** | 更新记录页可按版本号、标题、内容搜索 |
| **首次登录** | 不会弹出历史版本的累积提醒，仅提示登录后的新版本 |

> 更新内容由运维在**确认发版**后写入：平时变更先累计在待发列表，确认发版后才出现在「更新记录」并触发登录未读提醒。普通用户 **仅可查看**，不可编辑。

### 运维说明（待发 → 发版）

1. 功能调整后将变更记入 `server/releases/_pending.json`（可用 `npm run record-change`）
2. 未确认发版前，用户端看不到这些内容
3. 确认发版后执行 `npm run publish-release`（默认 patch 版本 +1），写入正式版本并更新「更新记录」

## 13. 常用快捷操作

| 操作 | 入口 |
|------|------|
| 快速建临时任务 | 待办页「新建事项」或移动端底部「新建」 |
| 查看任务详情 | 我的待办 → 点击任务卡片 |
| 任务留言 / @ 提及 | 任务详情 → 留言讨论 → 输入 `@` 或 Ctrl+V 粘贴图片 |
| 粘贴任务附件图片 | 任务详情 → 附件区域点击后 Ctrl+V |
| 导出任务列表 | 我的待办页顶部「导出」 |
| 处理辅助性协办 | 任务详情 → 协办审批区 → 填写时段并提交 |
| 查看全公司项目 | 侧边栏「项目」 |
| 查看系统更新 | 更多 →「更新记录」 |
| 查看操作日志 | 更多 →「操作日志」 |
