# 执行人员 操作手册

> 适用角色：`staff`（执行人员）  
> 权限范围：本人负责/协办的任务可编辑；可 **新建项目**（本人为负责人/创建人时可管理）；其他项目只读浏览

---

## 1. 您能看到什么

- **项目管理**：可查看 **全公司** 所有未归档项目（列表、详情、项目下 **全部** 任务树）
- **任务详情**：可打开任意项目下的任务详情，**只读** — 无编辑/转办/驳回等（除非您是负责人/协办人/项目负责人）
- **编辑权限**：本人为 **主负责人/协办人/项目负责人/创建人/项目团队成员** 的任务可编辑
- **我的待办**：本人末级任务；辅助性协办审批入口在任务详情内（原协办专区能力保留）
- **操作日志**：在 **项目详情 → 项目变更** 中查看与该项目相关的修改记录

**看不到**：其他同事的临时任务（无项目归属）、团队管理、人员档案、归档管理、数据导入（均在「更多」且本角色无入口）。

---

## 2. 菜单说明

| 菜单 | 用途 |
|------|------|
| **我的待办** | 每日入口：告警 + 本人末级任务 |
| **项目** | 查看全公司项目；可新建项目（本人为创建人/负责人时可管理） |
| **新建事项**（待办页） | 创建临时事项（可选挂到有权管理的项目） |

---

## 3. 典型操作流程

### 3.1 每日开始：我的待办

1. 打开恒慧管 → **我的待办**
2. 关注顶部红色 **逾期**、黄色 **临期** 提示
3. 点击任务卡片进入详情处理；待填时段的辅助协办在详情中审批区处理

### 3.2 处理常规任务

1. **我的待办** → 点击任务
2. 查看：所属项目、计划时间、任务描述、上级任务路径
3. 按状态操作：

| 当前状态 | 您可执行 |
|----------|----------|
| 待开始 | **开始**（记录实际开始日期）；若 **阻塞中** 则须等待前置完成 |
| 进行中 | 拖动 **进度滑块**；无子任务时可 **完成** |
| 阻塞中 | 在详情 **更多信息** 查看依赖；前置完成后自动解除 |
| 已驳回 | 查看驳回原因 → **更多 → 编辑** 整改 → 重新 **开始** |

4. 上传 **任务附件**（交付截图、文档等）：点击附件区后 **Ctrl+V** 可粘贴图片
5. 在 **留言讨论** 中与项目负责人、协办人沟通：输入 **@** 提及对方，**Ctrl+V** 粘贴过程截图
6. 完成后点击 **完成** → 系统通知 **项目负责人** 与 **您的直属上级**

> 若任务有 **未完成的子任务**，您不能直接完成上级任务（需先完成子任务或由经理处理）。

### 3.3 协办任务

#### 告知性协办

- 在 **我的待办** 列表与任务详情查看相关任务，跟进进展即可
- **不计入** 您的工时饱和度

#### 辅助性协办

1. 收到钉钉 **协办邀请** 后，打开任务详情
2. 填写可协助的 **开始/截止日期** 与说明 → **提交审批**
3. 任务负责人或项目负责人 **同意** 后，协助工时计入您当周饱和度
4. 审批结果会通过钉钉通知您

### 3.4 任务留言与 @ 提醒

当您为任务 **负责人** 或 **协办人** 时：

1. 任务详情 → **留言讨论**
2. 输入说明文字；输入 **@** 选择负责人、协办人或项目负责人
3. **Ctrl+V** 粘贴截图作为过程说明（发送前点击缩略图可放大预览）
4. 发送后，被 @ 且已绑定钉钉 userid 的人员会收到 **留言提及** 通知

> 留言图片不会进入任务附件列表；正式交付物请仍上传到 **任务附件**。

### 3.5 编辑自己的任务

1. 任务详情 → **编辑**
2. 可修改：标题、描述、计划开始、预计工时、进度、状态等
3. **必须填写修改原因** → 保存

> 不能修改负责人（需请经理 **转办**）；不能新建项目下主任务/子任务（需请经理拆分）。

### 3.6 任务转办

当无法继续执行时：

1. 任务详情 → **转办**
2. 选择本部门或其他部门同事（按列表可选范围）
3. **必填转办原因** → 确认
4. 新负责人收到钉钉通知

> 仅 **本人负责** 的任务可发起转办（经理/管理员也可代转）。

### 3.7 快速建临时任务

适用于不在正式项目内的零星工作：

1. 待办页 **新建事项**（或手机底栏「新建」）
2. 填写标题、描述、负责人（默认自己）
3. 可选填计划时间与优先级
4. 保存 → 出现在任务中心「临时任务」标签

> **表单提报**（钉钉 AI 表格生成的临时任务）：入库时预计工时为 0、不计入饱和度；点 **开始** 或 **完成** 时须填写预计工时，之后按与常规任务相同规则计入当周饱和度。完成时可确认通知提单人（无需理由）；**暂停**时可确认通知提单人，须写明暂停原因。自建临时任务创建时可直接填预计工时。

### 3.8 新建项目

执行人员也可创建项目，用于个人或小范围事项管理：

1. **项目管理** → **新建项目**
2. 填写名称、描述；默认 **项目负责人** 与 **创建人** 为您本人
3. 保存后可进入项目详情，添加任务、上传文档（须为项目负责人/创建人）

> 若将项目负责人设为他人，您仍为 **创建人**，可继续管理该项目；编辑/归档他人负责且非您创建的项目仍为只读。

### 3.9 作为项目团队成员

若您被项目负责人指定为 **团队成员**，对该项目下任务可：

| 可以做 | 不可以做 |
|--------|----------|
| 编辑任务、更新进度、上传附件、留言 | 添加主/子任务 |
| 查看项目全部任务树 | 驳回、转办（须项目负责人/管理员） |
| | 归档项目、上传项目文档 |

> 团队成员权限 **仅限被指定的项目**；其他项目仍为只读浏览。

### 3.10 查看项目背景

1. **项目管理** → 点击项目
2. 可查看项目描述、进度、全部任务树
3. 对 **非本人负责/创建** 的项目：不能添加任务、归档或上传项目文档

### 3.11 导出本人任务

**任务中心** 页顶部 → **导出** → 下载 Excel 任务清单（仅本人负责/协办任务）。

## 4. 权限边界

| 可以做 | 不可以做 |
|--------|----------|
| 更新 **本人负责** 的任务进度与状态 | 编辑/归档 **他人负责** 的项目 |
| 新建项目（默认为自己为负责人/创建人） | 删除项目（仅总经理/管理员） |
| 编辑本人任务（填修改原因） | 在 **他人项目** 下添加主/子任务 |
| 在 **本人负责/创建** 的项目下添加任务、归档、上传文档 | **驳回** 他人任务 |
| 转办本人任务 | **作废** 任务（需项目负责人/管理员） |
| 上传本人任务附件（含 Ctrl+V 粘贴图片） | 团队管理、人员档案、数据导入 |
| 作为 **项目团队成员** 编辑指定项目内任务 | 修改 **他人项目** 文档 |
| 辅助性协办：填写时段并提交审批 | 审批他人协办时段（须负责人/项目负责人） |
| 创建 **临时任务** | 编辑他人任务（非本人相关时） |
| 导出本人任务列表 | — |
| 在项目管理中 **只读** 查看全公司项目与他人任务 | — |

## 6. 工时、截止日期与依赖

- 填写 **计划开始** + **预计工时** 后，系统自动计算 **计划截止日期**（按公司工作日历）
- 若经理分配了 **每日投入工时**（如带教每天 2h），截止日按该值推算，而非满日 8.5h
- **开始执行** 后记录实际开始日；**标记完成** 后自动计算 **实际工时**
- 子任务若由经理分配，工时不能超过上级任务剩余额度
- **辅助性协办** 已批准时段按周分摊计入饱和度；**告知性协办** 不计入
- **跨项目依赖**：任务详情可查看 **依赖关系**；被阻塞时不能开始/完成，且 **不计入** 当周饱和度；解除阻塞后会收到钉钉通知

## 7. 常见问题

**Q：为什么我没有「添加任务」按钮？**  
A：仅 **项目负责人或创建人** 可在项目下添加任务。若该项目由他人负责且您非创建人，请联系项目负责人；您也可 **新建项目** 后自行拆分任务。

**Q：任务被驳回了怎么办？**  
A：查看详情顶部驳回原因 → 编辑修改 → 保存（填原因）→ 重新开始执行。

**Q：完成了为什么我没收到钉钉？**  
A：完成通知发给 **项目负责人** 和 **您的直属上级**，不发给执行人本人。若您兼项目负责人则会收到。

**Q：进度滑块拖不动？**  
A：可能该任务有未完成的子任务，或任务已结束/归档。请完成子任务或联系经理。

**Q：任务显示「阻塞中」无法开始？**  
A：存在未完成的 **前置依赖**。打开任务详情 → **依赖关系** 查看前置任务，等待其完成或联系经理/项目负责人调整。

**Q：我的待办为什么看不到同事的任务？**  
A：我的待办仅显示 **本人** 负责/协办任务。查看同事进度请进入 **项目** 找到对应项目。

**Q：临时任务和常规任务有什么区别？**  
A：临时任务不挂项目，您可自行创建；常规任务由经理在项目下分配，计入项目进度。

**Q：留言 @ 了经理没收到钉钉？**  
A：确认经理人员档案已绑定 userid，且您 @ 的是该任务上的负责人、协办人或项目负责人。

**Q：被设为项目团队成员，为什么还不能添加子任务？**  
A：团队成员可编辑/留言/上传附件，但 **拆分任务** 仍须项目负责人或总经理/管理员操作。
