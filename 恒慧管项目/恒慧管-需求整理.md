### 1.3 近期新增且已落地（2026-06）

#### R-001 全员可查看全公司项目（查看/编辑分离）

**规则：**

- **查看（canView*）**：人员档案中 **所有登录用户** 均可：
  - 项目管理列表 → 全公司未归档项目
  - 项目详情 → 项目下 **全部** 任务树
  - 任务详情弹窗 → **只读**（附件、留言可看）
- **编辑（canEdit* / canManage*）**：仅以下情况可改：
  - 总经理/管理员：全部
  - 部门经理：本人为项目负责人/创建人的项目及下属任务
  - 执行人员：本人为负责人/协办人/项目负责人/创建人
- **不变**：**无项目归属的临时任务** 仍仅相关人可见；编辑权限仍按负责人/协办/经理范围
- **任务页（2026-07-22）**：默认展示可见范围内全部任务；可切换「与我相关 / 我发起的 / 我处理的」；工作台「我的待办」仍只汇总本人事项
- **sync 安全**：非管理员 sync 时只上传本人可编辑的项目/任务，避免冲掉服务端全量数据

**已实现位置：**

- 前端：`canViewAllProjects` / `getViewableProjects` / `canViewTask`（`恒慧管.html`）
- 后端：`server/src/utils/projectAccess.js` + `data.js` 的 `filterByRole`
- 验收脚本：`node server/scripts/test-project-access.js`

**部署注意：** 若用户仍只看到少量项目，多为 **线上服务端未更新** 或 **数据库曾被旧版 sync 裁减** → 重启后端、项目管理页点 ↻ 刷新；必要时从 `server/data/backups/` 恢复。

#### R-006 操作面 Lite（仅 UI，业务逻辑不变）（2026-07）

**做了什么：**

- 首页「我的待办」合并原工作台 + 任务中心；导航为 **待办 / 项目 / 新建 / 更多**
- 2026-07-21 UI 路径 A（P0–P6）：色板对齐扣子 DESIGN；导航待办/项目/团队/更多；待办紧急度与行内快操作；详情三 Tab；项目卡片；待办看板拖拽；团队饱和度换皮；**业务逻辑与算法不变**
- 2026-07-22：对齐扣子模板壳层——主导航 **工作台 / 项目 / 任务 / 团队**（工作台第一且默认落地）；工作台含四色统计卡、新建事项、待办卡片、最近动态；本系统多出入口（绩效/档案/归档/导入/更新记录）迁入侧栏底部 **设置齿轮下拉**；项目计划、里程碑、变更记录统一在 **项目详情**；**任务页默认展示可见范围内全部任务**（可筛与我相关/我发起/我处理），工作台「我的待办」仍只汇总本人事项
- 2026-07-22：**工作汇报**：设置入口；按本人任务自动生成钉钉对齐的日报/周报全文（含方案意见），分栏复制与保存（不提交上级）
- 任务详情主操作：开始 / 完成 / 留言；转办、驳回、作废、编辑、拆子任务进「更多」
- 编辑表单默认精简；协办 / 工时 / 依赖折进「高级选项」；强制修改原因保留
- 新建事项三字段（标题、负责人、截止日期），可选挂项目；更深任务树仅提示建议两级，**不限制保存**
- 导入、饱和度、人员档案等入口在「设置」下拉（原「更多」），计算与审批规则未改

**未改：** 权限矩阵、协办双模式与审批、饱和度、依赖阻塞、强制改因、钉钉通知

**文档：** `server/docs/操作手册/00-通用说明.md`、`README.md` 已同步 Lite 菜单说明

#### 绩效管理（V4 考核表入库）

- 「设置」→「绩效管理」：内置项目经理通用绩效考核表 V4；**仅显示本人**考核表
- 以**月度**为单位，顶部下拉切换月份；无该月考核单时自动创建
- 证据按「周期_姓名_指标_类型_序号」标准命名；可选同步到关联项目文档（`source: performance`）
- 权限：仅 `.env` 中 `PERFORMANCE_ADMIN_USER_ID` / `PERFORMANCE_ADMIN_USER_NAME` 指定的一人可访问

```

任务中心列表        := 默认可见范围内全部末级任务（临时任务仍仅相关人）；可筛与我相关 / 我发起 / 我处理；工作台「我的待办」仍仅本人
```
